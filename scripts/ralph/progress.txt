## Codebase Patterns
- pnpm check = biome lint/format only (exit 0 = pass). tsc must be run separately in apps/renderer.
- cargo check -p mrunner validates all Rust changes.
- All frontend user-facing text must use t() with keys in both en.json and pt-BR.json.
- UserPreferences stored in ~/.config/mrunner[-dev]/preferences.json, validated with UserPreferencesSchema (zod).
- Tauri commands registered in apps/launcher/src/lib.rs invoke_handler array.
- @mrunner/ui exports from packages/ui/src/index.ts; ai-elements via @mrunner/ui/ai-elements/*.
- Config file I/O is now centralized in apps/renderer/src/lib/config-file.ts: getConfigFilePath, ensureConfigDir, readConfigFile<T>, writeConfigFile<T>. Import from @/lib/config-file.
- Keyboard shortcuts types in apps/renderer/src/core/types/shortcuts.ts: Hotkey { modifiers: Modifier[], key: string }, ShortcutConfig { id, type, context, hotkey, action, enabled, isCustom }.
- Global shortcuts sync to Rust via invoke('sync_global_shortcuts') — only type=global shortcuts.
- Internal shortcuts now use getHotkeyManager().register() from @tanstack/react-hotkeys — only type=internal, context=launcher. HotkeysProvider wraps app in main.tsx.
- HotkeyPicker at apps/renderer/src/components/shortcuts/hotkey-picker.tsx uses useHotkeyRecorder; recording starts on click, cancels on blur/Escape. Display via formatForDisplay(hotkeyToTanStack(value)).
- @tanstack/react-hotkeys installed (v0.4.0). Adapter at apps/renderer/src/lib/hotkey-adapter.ts: hotkeyToTanStack(Hotkey)->string, tanStackToHotkey(string)->Hotkey. Mod=Super on Linux, Mod=Meta on Mac.
- Shortcut consumers: App.tsx, settings-sheet.tsx, setup.tsx, shortcut-item.tsx.
- command.ts (347 lines) has types (lines 5-117), guards (lines 153-183), Zod schemas (lines 185-270), plugin functions (lines 306-346). 19 files import from it via @/commands/types or @/core/types.
- useBookmarks has 6 responsibilities: CRUD, interactions (open/copy), tag management, state, Tauri invocations, parseQuery utility.
- Pure utilities extracted from hooks go in apps/renderer/src/lib/. Module-level functions are stable refs — no useCallback needed when re-exporting imports.
