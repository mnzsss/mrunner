## Codebase Patterns
- Rust modules use `pub mod` in lib.rs when types need to be accessible from Tauri commands in the same file
- All Rust types that cross the Tauri IPC boundary use `#[serde(rename_all = "camelCase")]` for frontend interop
- Cargo workspace dependencies (serde, serde_json, dirs) are referenced via `dep.workspace = true` ŌĆö no version needed
- `cargo check -p mrunner` is the quality gate for Rust changes (not `cargo build`)
- tsup bypasses `noEmit: true` in tsconfig.json (uses esbuild internally) ŌĆö safe to use even with type-only tsconfig
- tokio `macros` feature cannot be used (tokio-macros 2.6.0 not in local registry); use `tokio::spawn` for concurrency instead of `tokio::join!`
- tokio async process I/O: take stdout/stderr handles from child BEFORE entering timeout so child stays owned for kill(); spawn tasks for concurrent reads
- CommandAction union is in `apps/renderer/src/core/types/command.ts`; new action types must be added there AND in the switch in `use-commands.ts` to avoid `never` error
- `core/types/index.ts` and `commands/types.ts` are re-export barrels ŌĆö new types/guards must be added to both
- Plugin action dispatch: use `executePluginAction` from `apps/renderer/src/lib/execute-plugin-action.ts`; navigation callbacks (push) passed as options to keep utility stateless
- Settings tabs wired in `settings-sheet.tsx`: TABS const + label render + Alt+N range + SheetBody block + i18n key; `UserPreferences` interface and `UserPreferencesSchema` must stay in sync
